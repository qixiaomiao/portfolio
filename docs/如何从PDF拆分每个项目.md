# 如何从 PDF 里拆分每个项目

你的作品集 PDF 里通常是一个项目占几页、带标题和若干张图。要对应到网站上的「一个项目 = 一个详情页 + 多张图」，按下面步骤做即可。

---

## 一、在 PDF 里怎么「数」出每个项目

1. **打开 PDF，从头翻到尾**
   - 看**大标题 / 章节名**（如「项目一」「XX 产品改版」「某数据平台」），每个大标题算一个项目。
   - 如果 PDF 没有明显标题，就按**内容主题**分：换了一个产品、换了一个业务，就另起一个项目。

2. **记下每个项目的范围**
   - 在纸上或表格里列出来，例如：
     - 项目 1：第 3–7 页，标题「XX 数据看板」
     - 项目 2：第 8–12 页，标题「YY AI 助手」
     - …
   - 这样你就知道：**一共有几个项目**、**每个项目从哪页到哪页**。

3. **每个项目要准备的信息**
   - **标题**：和 PDF 里一致，或稍作精简。
   - **一句话摘要**：背景 + 目标 + 你的角色/结果（1–2 句）。
   - **角色**：如「产品设计」「数据可视化」「用户研究」等。
   - **年份**：如 2024、2023。
   - **标签**：3–5 个关键词，如「AI 产品」「Dashboard」「B 端」。
   - **图片**：该项目在 PDF 里出现的图，按顺序挑出来（见下一节）。

---

## 二、每个项目的图怎么「分离」出来

PDF 里的图不能直接给网站用，需要**导出成一张张图片文件**。有两种常用方式。

### 方式 1：从 PDF 里导出图片（推荐）

- **Mac**：用预览打开 PDF → 左侧缩略图里选中某一页 → 菜单「文件」→「导出」→ 格式选 **PNG** 或 **JPEG**，导出为一张图。  
  或使用「预览」的「编辑」→「选择全部」→ 复制，再粘贴到其他软件里另存为图片。
- **Windows**：可用 Adobe Acrobat、或浏览器打开 PDF 后截图/另存为图片。
- **在线**：搜索「PDF 转图片」，把单页或选中的页转成 PNG/JPG 后下载。

对**每个项目**：
- 把该项目占的那几页，**一页导出一张图**（或一页里有多张图的话，按需要截成多张）。
- 按顺序命名，例如：`01.jpg`、`02.jpg`、`03.jpg`……

### 方式 2：截图

- 打开 PDF 到对应页，用系统截图（如 Mac 的 `Cmd+Shift+4`）截下整页或关键区域。
- 同样按顺序命名：`01.jpg`、`02.jpg`……

---

## 三、图片在项目里放哪里、怎么命名

网站约定了一个**固定文件夹结构**，你只要按这个放，改数据即可。

1. **在项目根目录下找到 `public` 文件夹**（没有就新建一个）。

2. **在 `public` 里建一个子文件夹 `projects`**：
   ```
   public/
   └── projects/
   ```

3. **每个项目单独一个子文件夹**，文件夹名用**英文或拼音**（会变成详情页网址），例如：
   - `public/projects/data-dashboard/`   → 对应网址 `/projects/data-dashboard`
   - `public/projects/ai-assistant/`
   - `public/projects/xx-redesign/`

4. **把该项目的图片都放进这个文件夹**，命名建议：
   - **封面图**（列表页卡片用）：`cover.jpg` 或 `cover.png`
   - **详情页多张图**（按顺序）：`01.jpg`、`02.jpg`、`03.jpg`……

示例：

```
public/
└── projects/
    ├── data-dashboard/
    │   ├── cover.jpg    ← 列表页卡片图
    │   ├── 01.jpg       ← 详情页第 1 张
    │   ├── 02.jpg
    │   └── 03.jpg
    ├── ai-assistant/
    │   ├── cover.jpg
    │   ├── 01.jpg
    │   └── 02.jpg
    └── xx-redesign/
        ├── cover.png
        └── 01.png
```

这样，**一个物理项目 = 一个文件夹 + 多张图**，和 PDF 里「一个项目几页」一一对应。

## 四、在网站里怎么对应「每个项目」

打开项目里的 **`data/projects.ts`**，里面是一个数组，**数组里的一项 = 一个项目**。

对你在 PDF 里数出来的**每一个项目**，在 `projects` 数组里加（或改）一条，例如：

```ts
{
  slug: "data-dashboard",        // 和 public/projects/ 下的文件夹名一致
  title: "XX 数据看板设计",
  summary: "一句话描述背景、目标和你的贡献。",
  role: "产品设计 / 数据可视化",
  tags: ["数据看板", "B 端", "可视化"],
  year: "2024",
  image: "/projects/data-dashboard/cover.jpg",
  images: [
    "/projects/data-dashboard/01.jpg",
    "/projects/data-dashboard/02.jpg",
    "/projects/data-dashboard/03.jpg",
  ],
},
```

- **slug**：必须和 `public/projects/` 下的**文件夹名**一致，且用英文/数字/连字符（不要空格、中文）。
- **image**：列表页卡片用的图，一般用该项目的 `cover.jpg`。
- **images**：详情页从上到下展示的图，顺序和 PDF 里一致即可。

你从 PDF 拆出几个项目，就建几个文件夹、在 `projects.ts` 里写几条；**每个物理项目 = 一个文件夹 + 一条数据**。

---

## 五、小结：拆分流程

| 步骤 | 做什么 |
|------|--------|
| 1 | 打开 PDF，按标题或主题数出「有几个项目」，记下每项占的页码。 |
| 2 | 每个项目：写标题、摘要、角色、年份、标签（可先写在文档里）。 |
| 3 | 每个项目：把对应页导出/截图为 01.jpg、02.jpg…，选一张当 cover.jpg。 |
| 4 | 在 `public/projects/` 下为每个项目建一个文件夹（英文名），把图放进去。 |
| 5 | 在 `data/projects.ts` 里为每个项目写一条，slug、image、images 按上面填。 |

做完后运行 `npm run dev`，在「项目」列表点进任意一项，就会打开该项目的详情页，并看到你放的图。  
如果你愿意，我可以根据你 PDF 的目录结构，帮你列一版「项目清单」模板（只填标题和 slug），你按表填完再贴进 `projects.ts` 即可。
